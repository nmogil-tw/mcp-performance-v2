Project Overview

The MCP-TE Benchmark is designed to measure the efficiency gains when AI coding agents utilize structured context protocols (like MCP) compared to traditional development methods. As AI coding assistants become more integrated into development workflows, understanding how they interact with APIs becomes increasingly important for optimizing developer productivity and cost.

Testing Methodology

Test Environment

Tests are conducted using Cursor IDE in "auto-run" mode, allowing AI agents to execute actions autonomously. We use Claude 3.7 Sonnet as our primary AI model and Twilio APIs as our testing platform.

Metrics Collection

We automatically track and extract the following metrics from each test session:

  • Duration: Time taken to complete tasks
  • API Calls: Number of API calls made
  • Interactions: Number of exchanges between user and AI
  • Token Usage: Input and output tokens used
  • Cache Performance: Reads and writes to internal cache
  • Cost: Estimated cost based on token usage

Test Groups

Control

Traditional method using web search and terminal capabilities to complete API tasks. The Control approach requires manually looking up documentation and writing code from scratch.

MCP

Model Context Protocol approach that provides AI coding agents with direct access to API functions. MCP enables AI to understand API structure without external documentation.

Benchmark Tasks

Task 1: Purchase a Canadian Phone Number

Search for and purchase an available Canadian phone number (preferably with area code 416).

Challenge: Requires understanding available API endpoints, search parameters, and purchase flow.

Task 2: Create a Task Router Activity

Create a new Task Router activity named "Bathroom".

Challenge: Requires understanding Twilio's TaskRouter concepts and API structures.

Task 3: Create a Queue with Task Filter

Create a queue with a task filter that prevents routing tasks to workers in the "Bathroom" activity.

Challenge: Requires understanding of filter syntax and queue configuration parameters.

Insights & Implications

API Efficiency

MCP significantly reduces API calls (63.8% reduction) by providing structured access to API functions. This suggests MCP helps AI assistants navigate APIs more efficiently, reducing trial-and-error approaches.

Time Efficiency

Despite fewer API calls, the overall time efficiency shows only a modest improvement (-1.4%). This suggests that while MCP streamlines API interaction, other factors like planning and context processing still influence overall task duration.

Developer Experience

The reduction in interactions (-16.7%) indicates that developers need to provide less guidance when MCP is available, potentially improving workflow and reducing context switching.

Future Directions

These results suggest that MCP provides the most value in complex API ecosystems where traditional documentation is extensive or difficult to navigate. Future benchmarks could explore this hypothesis across different API complexities.
